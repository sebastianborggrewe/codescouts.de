Änderungen an Dateien nachzuvollziehen ist immer so eine Sache. Man kann natürlich nach jeder wichtigen Änderung eine Datei kopieren und hat schließlich verschiedene Dateien: Final1, Final2, Final3...

Einfacher ist es GIT zu benutzen. Hier arbeitet man weiter an ein und derselben Datei und kann Änderungen nach Belieben speichern und rückgängig machen. Sebastian gibt dir eine kurze Einführung, die alle wichtigen Basics von GIT abdeckt, damit du nie mehr fluchst, wenn die Undo-Funktion nicht mehr klappt!
